# 📄 PaperLens 프로젝트 기획서

## 📌 프로젝트명

**PaperLens**

> 논문을 '읽는' 것이 아니라 '탐색'하게 만든다

---

## 1. 🎯 프로젝트 목표

PDF 논문을 업로드하고,
사용자가 입력한 키워드를 **정확한 의미 기반으로 검색**하며,
결과를 **위치 이동 + 하이라이트 + 번역 기능**으로 제공하는 웹 애플리케이션 구축

### 3.2 🔍 키워드 검색

#### ✔ 기본 기능

* 키워드 입력
* 문서 내 등장 위치 표시
* 등장 횟수 표시

#### ✔ 고급 기능 (핵심 차별화)

##### 문제

* "ai" 검색 시 claim, said 등 불필요한 결과 포함

##### 해결 방식

* 문자열 검색이 아닌 **토큰 기반 + 의미 기반 검색**

---

#### ✔ 검색 레벨

**Level 1: Exact Match**

* 완전 일치 단어만 검색
* 부분 문자열 제거 (대소문자 무시 옵션)

**Level 2: AI Semantic Search**

* 의미 기반 검색
* AI ↔ Artificial Intelligence 동일 처리
* **문장/청크 단위 임베딩 비교** (토큰 단위 X)

---

#### ✔ 처리 과정

1. PDF 업로드 시 텍스트 추출
2. 문장 단위로 분할 + 위치 정보 매핑
3. (Semantic 모드) 업로드 시 문장별 임베딩 사전 생성 및 캐싱
4. Exact Match: 단어 단위 인덱스에서 검색
5. Semantic Search: 키워드 임베딩 vs 사전 생성된 문장 임베딩 유사도 비교

## 8. 🔥 핵심 알고리즘

### ✔ Exact Match 검색

```ts
function exactSearch(sentences: Sentence[], keyword: string): SearchResult[] {
  const regex = new RegExp(`\\b${keyword}\\b`, 'gi')
  return sentences
    .filter(s => regex.test(s.text))
    .map(s => ({ page: s.page, position: s.position, text: s.text }))
}
```

---

### ✔ Semantic Search

```ts
// 1. 업로드 시: 문장별 임베딩 사전 생성
const sentenceEmbeddings = await embedBatch(sentences) // text-embedding-004

// 2. 검색 시: 키워드 임베딩 1회 생성 → 사전 임베딩과 비교
const keywordEmbedding = await embed(keyword)
const results = sentenceEmbeddings
  .map(s => ({ ...s, score: cosineSimilarity(keywordEmbedding, s.embedding) }))
  .filter(s => s.score > THRESHOLD)
  .sort((a, b) => b.score - a.score)
```
